About

Engineering ML for the realities of clinical data.

I'm a Machine Learning Engineer working at the intersection of deep learning and healthcare. My focus is building models that hold up under the real constraints of clinical data — class imbalance, demographic disparity, and the need for predictions a clinician can interpret and trust.

My work spans machine learning, deep learning, and computer vision, applied across clinical data from structured records to medical images — with fairness-aware modeling and interpretability running throughout. Across seven first-authored publications, several of these models don't stop at the paper: I've deployed them as interactive Streamlit applications, and I build with deployment in mind, from containerization to CI/CD.

My background spans graduate degrees in engineering and health sciences, which is where the dual instinct comes from: rigorous about the modeling, grounded in the clinical question it's meant to answer.

012026

End-to-End ML CI/CD Pipeline

DockerGitHub ActionsDocker Hub
Problem

Breast-cancer prediction models needed a path from code to deployment that was reproducible and safe to automate.

Approach

Built a full CI/CD pipeline with Docker and GitHub Actions — container images published to Docker Hub, with quality gates that halt deployment when evaluation metrics fall below defined thresholds.

Result

Automated, reproducible deployment with metric-gated releases that block regressions before they ship.

022025

Graph Neural Network for Coronary Artery Disease Mortality Prediction

Graph Neural NetworksPyTorchGNNExplainerStreamlit
Problem

Predicting mortality in coronary artery disease where relationships between patients carry signal that flat models miss.

Approach

Deployed a deliberately lightweight graph neural network — fewer than 1,000 parameters — in PyTorch, with GNNExplainer for interpretability and a live Streamlit interface.

Result

93.0% recall, a +12% gain over the best baseline, from a model small enough to interrogate.

Live app
032025

Ensemble Classification for Pharmacogenetic Testing Uptake

Ensemble MLscikit-learnEHRImbalanced Learning
Problem

Predicting pharmacogenetic testing uptake from imbalanced electronic health record data, where single classifiers underperformed.

Approach

Built and validated a weighted-average ensemble pipeline — benchmarking multiple classifiers and combining the top performers on imbalanced EHR data.

Result

79.6% AUC · 76.5% recall, outperforming every individual baseline.

042024

Large-Scale Retrospective Pharmacoepidemiological Analysis

SASSQLPropensity Score MatchingLogistic Regression
Problem

Estimating the effect of psychostimulant use on opioid-use-disorder outcomes across a decade of claims data.

Approach

Engineered a multi-table analytical pipeline in SAS and SQL — joining and transforming claims data to build analytical samples, then applying propensity score matching and logistic regression.

Result

Decade-scale causal estimates from a reproducible observational-data pipeline.

052023

Fairness-Aware Classification for Opioid Use Disorder Prediction

Neural NetworksBias MitigationEquality of Odds
Problem

Opioid-use-disorder predictions that perform well overall can still carry large disparities across demographic groups.

Approach

Developed an equality-of-odds bias-mitigation algorithm and a fairness-aware weighted majority voting classifier.

Result

92.7% recall · 90.2% accuracy while measurably reducing demographic disparities.

062022

Psoriasis Detection & Treatment Recommendation

CNNTensorFlowResNetV2 EnsembleStreamlit
Problem

Detecting psoriasis and classifying its subtype from images, then turning that into an actionable treatment recommendation.

Approach

Built and deployed a hierarchical image-classification pipeline using a weighted-average ensemble of ResNetV2 networks, integrated with a multi-criteria decision-analysis module.

Result

91.9% binary accuracy · 93.3% on 7-class subtyping, with automated treatment recommendations.
